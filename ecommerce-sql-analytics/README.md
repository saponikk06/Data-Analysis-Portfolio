# Анализ воронки продаж и удержания пользователей интернет-магазина «Накарабине»

Данный проект направлен на изучение поведения пользователей и ключевых метрик интернет-магазина одежды и снаряжения «Накарабине». В основу легли бизнес-правила и логика, спроектированные мной ранее в рамках системного анализа продукта.

## 🎯 Цели проекта
* Построение воронки продаж для выявления этапов с наибольшим оттоком пользователей.
* Проведение когортного анализа для оценки удержания (Retention) клиентов.
* Проектирование и наполнение демонстрационной базы данных для аналитических задач.

## 🛠 Технологический стек
* **СУБД:** PostgreSQL 16.
* **SQL-инструментарий:** Оконные функции (`FIRST_VALUE`), обобщенные табличные выражения (CTE), агрегатные функции.

## 🗂 Модель данных
Схема данных реализована в PostgreSQL и состоит из трех связанных таблиц:
* **users** — информация о пользователях (ID, дата регистрации, регион).
* **orders** — данные о транзакциях (сумма заказа, статус оплаты, дата).
* **shop_events** — логи действий на сайте для анализа воронки (просмотры, добавления в корзину, оформление заказа).

## 📈 Ключевые отчеты

### 1. Воронка продаж (Conversion Funnel)
Для расчета конверсии использовались оконные функции, что позволило определить процент прохождения каждого этапа относительно первого шага («view»).
* **SQL-скрипт:** `funnel.sql`.
* **Результат:** На тестовых данных выявлено снижение конверсии на этапе оформления заказа (checkout) до 66.67%.

### 2. Когортный анализ (Retention Analysis)
Реализован расчет вернувшихся пользователей (returning users) по месяцам их регистрации и совершения покупок.
* **SQL-скрипт:** `cohorts.sql`.
* **Результат:** Сформирована матрица активности пользователей по когортам на основе данных таблиц `users` и `orders`.
