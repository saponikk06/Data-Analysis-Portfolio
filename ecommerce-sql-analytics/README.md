# Анализ воронки продаж и удержания пользователей интернет-магазина «Накарабине»

[cite_start]Проект базируется на анализе работы интернет-магазина одежды и снаряжения «Накарабине», для которого ранее проектировались бизнес-правила и логика платежных интеграций.

## 🎯 Бизнес-задача
Выявить этапы воронки с наибольшим оттоком пользователей и проанализировать эффективность возврата клиентов (Retention). [cite_start]В проекте решается задача по поиску причин, по которым 5% пользователей не завершают целевой путь.

## 🛠 Технологический стек
* **SQL (PostgreSQL):** СТЕ, оконные функции, агрегатные функции.
* **Метрики:** Conversion Rate (CR), Drop-off Rate, Retention Rate.

## 🧪 Что было сделано (SQL задачи)
1. **Анализ воронки (Funnel Analysis):** Построение пути пользователя от просмотра товара до успешной оплаты.
2. **Анализ брошенных корзин:** Выявление среднего чека и состава заказов, которые не были оплачены.
3. **Когортный анализ (Retention):** Расчет удержания пользователей по месяцам первой покупки.

## 📈 Ключевые инсайты
* Выявлено «узкое горлышко» на этапе выбора способа доставки (отвал 3% пользователей).
* [cite_start]Точность прохождения пользовательского пути подтверждена на уровне 95%.
* Retention 2-го месяца для лояльных категорий (снаряжение) выше на 10%, чем в категории «одежда».
